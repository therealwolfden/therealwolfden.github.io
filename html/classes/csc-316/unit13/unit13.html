Multiway Search Trees: (2, 4) Trees

2-4 Trees

Overview:
  • Generalization of a binary seach tree (multiway).
  • Every node has at least 4 children.
  • All leaves (ends) are at the same depth.
  • Grows from the top of the tree.
Nodes:
  • 4-nodes are nodes that contain 3 keys and 4 children.
    • C1 < K1 < C2 < K2 < C3 < K3 < C4
  • 3-nodes are nodes that contain 2 keys and 3 children.
    • C1 < K1 < C2 < K2 < C3
  • 2-nodes are nodes that contain 1 key and 2 children.
    • C1 < K1 < C2
Searching within a 2-4 tree is the same as a normal binary tree but all keys within a node are checked.

put(k,v):
  • If an entry with the key already exists in the map, update the value associated with the key.
  • Add k to leaf may cause an “overflow” or a node with more than 3 keys.
    • To fix an overflow replace the current node with 2 nodes.
      • The new 3-node will have children C1, C2, and C3 and will store keys K1 and K2.
      • The new 2-node will have children C4, and C5 and will store key K4.
      • K3 will be passed to the parent which may also be split.
remove(k):
  • If k is found in a leaf node just remove the leaf node. If not replace k with it's inorder successor.
  • Replacing k may cause an "underflow" or node with 0 keys where the inorder successor was taken from.
    • To fix an underflow a transference or fusion of the sibling node.
    • If the sibling node has more than one key then a parent key is moved to the empty node and a sibling key transfers to parent.
    • If the sibling node only has one key then the two siblings merge and a parent key moves down to the the merged node. If the parent is now empty repeat on the parent.
Performance:
  • Height is O(log(n))
  • get is O(h) or O(log(n))
  • put is O(h) or O(log(n))
  • remove is O(h) or O(log(n))

(a-b) Trees

Overview:
  • Generalization of a 2-4 tree.
  • a and b such that 2 ≤ a ≤ b + 1 ⁄ 2.
  • Also described as a B-tree with degree/order of 4.
  • Each internal node has children a ≤ c ≤ b
  • All leaves have the same depth
B-Trees:
  • Best known method for maintaining a map in external memory.
  • Reduces the number of accesses to the external storage.
  • Select b such that b pointer fields and b-1 keys can all fit into a single disk block.
B+Trees:
  • Special case of B-Trees.
  • Keys at interior nodes appear without their associated value and may not even be part of the map. They only provide index to guide towards the leaf.
  • Mainly used for maps stored in external storage devices (Disk, CD-ROM).
