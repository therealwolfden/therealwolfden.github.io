Set and Disjoint Set ADTs

Sets

Sets are an unordered collection of elements that do not contain duplicates.
  • Supports efficient "membership tests" such as contains.
  • Elements are like keys in a map but without an associated value.
  • An intersection (S1∩S2) of sets is the values that are contained within both sets.
  • A union (S1∪S2) of sets is the combination of two sets with matches only being listed once.
  • A subtraction (S1-S2) of sets is the removal of matching values from S1.
  • See Table P for a comprehensive guide to the methods a Set must implement. Briefly, a set should have...
    • add(e)
    • remove(e)
    • contains(e)
    • addAll(T)
    • retainAll(T)
    • removeAll(T)
    • size()
    • isEmpty()

Disjoint Sets

Disjoint Sets are a fixed set U of items.
  • No insertions, deletions, or subtractions.
  • Set U is divided into disjoint sets S1,...,Sk
  • An intersection (S1∩S2) of sets is the values that are contained within both sets.
  • Si∩Sj = ∅ for i ≠ j
  • S1∪S2∪...∪Sk= U
  • See Table Q for a comprehensive guide to the methods a Disjoint Set must implement. Briefly, a disjoint set should have...
    • makeSet(e)
    • union(s, t)
    • find(e)

Up-Tree Data Structure

Up-Tree:
  • Tree structure with pointers pointing up the tree (children points to parents).
  • Forest of up-trees is a group of disjoint sets.
  • No bound on number of children a node may have.
  • Terminology
    • Tree Node: one item in set U.
    • Up-tree: a set Si.
    • Forest of Up-trees:universal set U.
    • Root Nodes:identify sets (instead of using Si).
    • Sets Si are Disjoint: each item x belongs to exaclty one up-tree.

Up-Tree Behaviors

makeSet(x):
  • Create a new node.
  • Initialize its parent field to itself to indicate the node is the root.
  • Initialize its count field to 1.
Algorithm makeSet(x)
Input an element x
Output the root node of a new set
return newNode(x)

find(p):
  • Follow parent pointers until the node is reached.
Algorithm find(p)
Input a node p
Output the root node of the set containing p
while parent (p) ≠ p do
p ← parent(p)
return p

union(S, T):
  • Makes the root of one set point to the root of the other.
  • Always make the root of the tree representing the smallest set point to the root of the largest set (balanced).
Algorithm union(S, T)
Input a root node S
a root node T
Output the root node of the combined set
if count(S) > count(T) then
count(S) ← count(S) + count(T)
parent(T) ← S
return S
else
count(T) ← count(S) + count(T)
parent(S) ← T
return T

Up-Tree Performance

Performance:
  • makeSet(x) is O(1).
  • union(S, T) is O(1).
  • find(p) is O(h).
    • O(n) is the worst-case.
    • O(log(n)) when using a balanced union method as discussed.
  • Height hfor a balanced tree is h ≤ log(n) or n ≥ 2h.

Path Compression:
  • After performing find, make all nodes along the path to the root children of the root.
  • Increases the cost of the current operation.
  • Decreases the cost of subsequent operations.
Algorithm PCFind(p)
Input a node p to find
Output the root node of the set containing p
if p ≠ parent(p)
parent(p) ← PCFind(parent(p))
return parent(p)
