Big-Oh

Big-Oh: given functions f(n) and g(n), it can be said that f(n) is O(g(n)) if there are integer constants c > 0 and n0 ≥ 1 such that f(n) ≤ c * g(n) and ∀n ≥ n0

Proving Big-Oh Using Definition
  1. Chooose n0 = 1
  2. Assume n ≥ 1
    • n ≥ 1 implies 1 ≤ n, n ≤ n2, n2 ≤ n3, ...
  3. Find a c such that f(n) ≤ c * g(n)
Proving Big-Oh using Limit Test:if f(n) and g(n) are monotone increasing and positive, then...
  • if limn→∞ f(n)⁄g(n) = 0, then f(n) is O(g(n)), but g(n) is NOT O(f(n))
  • if limn→∞ f(n)⁄g(n) = ∞, then g(n) is O(f(n)), but f(n) is NOT O(g(n))
  • if limn→∞ f(n)⁄g(n) = c, 0 < c < ∞, then f(n) is O(g(n)), and g(n) is O(f(n))

Big-Omega

Big-Omega: given functions f(n) and g(n), it can be said that f(n) is Ω(g(n)) if there are integer constants c > 0 and n0 ≥ 1 such that f(n) ≥ c * g(n) and ∀n ≥ n0

Big-Theta

Big-Theta: given functions f(n) and g(n), it can be said that f(n) is Θ(g(n)) if there are integer constants c' > 0, c'' > 0 and n0 ≥ 1 such that c'g(n) ≤ f(n) ≤ c''g(n) and ∀n ≥ n0

Limit Test

Using Limit Test: if f(n) and g(n) are monotone increasing and positive, then...
  • if limn→∞ f(n)⁄g(n) = 0, then f(n) is O(g(n)), and g(n) is Ω(f(n))
  • if limn→∞ f(n)⁄g(n) = ∞, then g(n) is O(f(n)), but f(n) is Ω(g(n))
  • if limn→∞ f(n)⁄g(n) = c, 0 < c < ∞, then f(n) is Θ(g(n)), and g(n) is Θ(f(n))

Analysis of Algorithms

Running time: is the time it takes for a program to successfully execute. It can depend on a variety of things such as...
  • Software includes the language and compiler
  • Programmer includes the experience and coding details
  • Hardware includes the machine used
  • Algorithm includes the approach to solving the problem
Theoretical Analysis:
  • Uses pseudocode instead of the implementation
  • Characterizes the running time as a function of input size n
  • Independent of the hardware/software environment
