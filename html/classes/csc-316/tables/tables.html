Table M: Tree Implementations
The following shows the advantages and disadvantages of various tree implementations.

Tree Type Advantages Disadvantages
Binary Search Trees
  • Easy to implement.
  • No guarantee of balance.
  • Height may be O(n), leading to O(n) worst-case runtimes for Map behaviors.
AVL Trees
  • Guaranteed O(log(n)) height for worst-case O(log(n)) runtimes for Map ADT behaviors.
  • Must use extra space as each position must keep track of its own height.
  • May require restructuring at EVERY LEVEL of the tree when removing.
Splay Trees
  • No need for positions to store any extra information.
  • Move to front heuristic.
  • Amortized cost for Map ADT behaviors is O(log(n)).
  • Worst-case height is still O(n), leading to O(n) worst case runtimes for Map ADT behaviors.
  • Require restructuring (splay) at every level of the tree on every operation.
(2,4) Trees & B-Trees
  • Guaranteed O(log(n)) height for worst-case O(log(n)) runtimes for Map ADT behaviors.
  • More difficult to implement than BSTs.
  • May require restructuring (split and fusion) at every level of the tree when putting and removing.
Red-Black Trees
  • Guaranteed O(log(n)) height for worst-case O(log(n)) runtimes for Map ADT behaviors.
  • O(1) restructuring.
  • Must use extra space for each node to keep track of its own color.
