Graph Abstract Data Type

Graphs

Graphs:
  • Representation of relationships between objects.
  • Consider it as a collection of vertices and edges.

Undirected Graphs:
  • Symmetric relationships.
  • Edges are represented by an unordered pair of vertices.
  • Relationships are described by {V1, V2}

Directed Graphs:
  • Asymmetric relationships.
  • Edges are represented by an ordered pair of vertices.
  • Relationships are described by (V1, V2)

Terminology:
  • Endpoints: two vertices joined by an edge.
  • Origin Endpoint (Directed): vertex from which the edge leaves.
  • Destination Endpoint (Directed): the vertex in which the edge enters.
  • Adjacent Vertices: two vertices that are endpoints of the same edge.
  • Incident Edges: an edge in which the vertex is a part of.
  • Outgoing Edges (Directed): edges in which the vertex is the origin endpoint.
  • Incoming Edges (Directed): edges in which the vertex is the destination endpoint.
  • Degree: the number of incident edges of a vertex.
  • In-degree: the number of outgoing edges of a vertex.
  • Out-degree: the number of incoming edges of a vertex.
  • Parallel Edges: two edges that have the same endpoint vertices (direction must match for directed graphs).
  • Self-Loop: an edge whose endpoints are the same vertex.
  • Simple Graph: a graph that does not contain self-loops or parallel edges.
  • Path: a sequence of vertices that contains edges between each adjacent vertex.
  • Cycle: a path in which the first vertex is also the last vertex.
  • Reachable: a vertex is reachable if a path can be formed from the current vertex to it.
  • Subgraph: a graph whose vertices and edges are subsets of vertices in a larger graph.
  • Spanning Subgraph: a subgraph containing all vertices of its larger graph.
  • Connected: for any two vertices a path can be found between the two vertices.
  • Weakly Connected: a graph in which an undirected path exists between any two vertices.
  • Strongly Connected: a graph in which a directed path exists between any two vertices.
  • Connected Component: a maximally connected subgraph of a larger graph.
  • Complete Graph: a graph in which an edge exists between every distinct vertex.
    • For an undirected graph where n is the number of vertices and m is the number of edges, m =
      n(n-1)
      2
      .
    • For a directed graph where n is the number of vertices and m is the number of edges, m = n(n-1).

Properties:
  • Given a simple undirected graph...
    • Σdegree(v) = 2m
    • If the graph is connected, m ≥ n - 1
    • m is O(n2).
  • Given a simple directed graph...
    • Σindegree(v) = Σoutdegree(v) = 2m
    • If the graph is weakly connected, m ≥ n - 1
    • m is O(n2).

Graph ADT

Graph ADT:
  • For a comprehensive list of all graph methods see Table R.
  • We can model a directed graph and an undirected graph in the same ADT.
  • Can be modeled with any of the following data structures:
    • Edge List Structure
    • Adjacency List
    • Adjacency Map
    • Adjacency Matrix

Graph ADT Data Structures

Edge List Structure:
  • Maintains an unordered list of vertices and a unordered list of edges.
  • Vertices and edges are location-aware.
  • Vertices have no references to edges.
  • Simple but not space efficient, O(n + m).

Adjacency List:
  • Each vertex maintains a incidence collection.
  • Each vertex contains the data, the outgoing edges, and the incoming edges.
  • Instead of iterating through a global list of edges, we can iterate through the vertex's list of incident edges.
  • Simple but not space efficient, O(n + m).

Adjacency Map:
  • Each vertex maintains a incidence collection.
  • Each vertex contains the data, the outgoing edges and the incoming edges both of which are stores in hash maps.
  • Let the opposite endpoint serve as the map key and the edge as the value.
  • Simple but not space efficient, O(n + m).

Adjacency Matrix:
  • List of edges represented as a 2D array.
  • Row/column represent the origin/destination endpoints.
  • The index of the row/column represents the corresponding vertex. Column 1 and row 1 both represent the same vertex.
  • Buckets store the pointer to the edge object or null if no edge exists.
