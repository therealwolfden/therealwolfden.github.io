Tree ADT

Trees

Trees are a directed, acyclic structure of linked nodes.
  • Directed means that there are one way links between nodes.
  • Acyclic means that no path wraps back around to the same node twice.
  • Can be defined as empty (null) or as a root node that contains data and zero or more children nodes.
  • Often used for folders and files on a computer.
  • Can be used to model AI decision trees
  • For a comprehensive list of Tree operations see Table K.

Nodes are an object containing a data value and zero or more children.
  • Internal nodes are a node with at least one child.

Root is the topmost node of a tree and has no parent node.

Leaf or external nodes are nodes that have no children.

Branch any internal node (not a root or a leaf).

Subtree is the tree of nodes reachable from the current node.

Height the length of the longest path from the root to any node.

Level/Depth is the length of the path from the root to a given node.

Tree Traversals

Traversals:
  • A method for visiting every node in a tree.
  • Creates a well-defined ordering of tree nodes.
  • Some operation can be performed when visiting each node.

Preorder Traversal:
  • A node is visited before its descendents.
  • An example would be printing a directory structure.
  • Root → Left Child → Right Child
  • T(n) is O(n)
Algorithm preOrder(T, p)
Input a tree, T
a position, p
if p is null then
return
visit p
for each child w in T.children(p)
preOrder(T, w)

Postorder Traversal:
  • A node is visited after its descendents.
  • An example would be computing space used by files in a directory and its subdirectories.
  • Left Child → Right Child → Root
  • T(n) is O(n)
Algorithm postOrder(T, p)
Input a tree, T
a position, p
if p is null then
return
for each child w in T.children(p)
preOrder(T, w)
visit p

Levelorder Traversal:
  • A node is visited from top to bottom, left to right.
  • T(n) is O(n)
Algorithm levelOrder(T, p)
Input a tree, T
a position, p
Q ← initially empty queue of nodes
if p is null then
return
Q.enqueue(p)
while NOT Q.isEmpty() do
q ← Q.dequeue()
visit(q)
for each child w in T.children(q)
Q.enqueue(w)

Binary Trees

Binary Tree:
  • Each internal node has at most two children.
  • Children of a node are ordered with left and right.
  • Some operation can be performed when visiting each node.
  • An empty tree is a binary tree.
  • A node with left and right subtrees, each of which is a binary tree, is a binary tree.
Inorder Traversal:
  • A node is visited after its left subtree and before its right subtree.
  • An example would be outputing keys in order for a binary search tree.
  • Left Child → Root → Right Child
  • T(n) is O(n)
Algorithm inOrder(T, p)
Input a tree, T
a position, p
if p is null then
return
inOrder(T, T.left(p))
visit(p)
inOrder(T, T.right(p))

Height:
  • The minimum possible height of a binary tree is O(log(n)).
  • The maximum possible height of a binary tree is O(n).
  • The maximum possible number of nodes on a given level is 2h.

Tree Classifications

Proper Binary Trees
  • Each non-leaf node has exactly two children.
  • Number of leaves = number of non-leaves + 1

Perfect Binary Trees
  • Each non-leaf node has exactly two children.
  • All leaves are the same depth.
  • A perfect binary tree is a proper binary tree but a proper binary tree is not always a perfect one.
  • Contains 2h+1 - 1 nodes.
  • Contains 2h leaves.

Complete Binary Trees
  • All leaves are the same depth.
  • All leaves are filled from left to right but does not neccessarily have to be full.
  • The number of node n in a complete binary tree is 2k ≤ n ≤ 2k+1.
  • Garunteed least height.
