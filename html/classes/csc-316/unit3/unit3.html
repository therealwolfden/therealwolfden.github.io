Analysis of Algorithms

Running time: is the time it takes for a program to successfully execute. It can depend on a variety of things such as...
  • Software includes the language and compiler
  • Programmer includes the experience and coding details
  • Hardware includes the machine used
  • Algorithm includes the approach to solving the problem
Theoretical Analysis:
  • Uses pseudocode instead of the implementation
  • Characterizes the running time as a function of input size n
  • Independent of the hardware/software environment

Comparison Based Sorting

Comparison Based Sorting is the organization into data structures by making comparisons between pairs of elements. It is efficient for smaller data sets and data that is already almost sorted.

Bogosort is an algorithm that checks to see if the data is already sorted and if it is not the data is randomly shuffled before repeating the process.
  • What input of data would minimize the repetitions (most efficient)?
    • When the input data is already sorted it will not repeat at all.
    • Only goes through the code once so T(n) = n - 1.
    • T(n) is O(n)
  • What input of data would maximize the repetitions (least efficient)?
    • Any data that is not sorted can be equally efficient.
    • Regardless of how the unsorted data is entered the randomness makes it so T(n) is unbounded and can theoretically last forever.

Bubble Sort goes through an array of data and goes pair by pair swapping data as necessary. This process is repeated until a full pass is completed without swapping.
  • What input of data would minimize the repetitions (most efficient)?
    • When the input data is already sorted it will not repeat at all.
    • Only goes through the code once so T(n) = n - 1.
    • T(n) is O(n)
  • What input of data would maximize the repetitions (least efficient)?
    • Data sorted in reverse sorted order as it would have to move data from one end to the other.
    • If the data is in reverse order then T(n) = n(n - 1) = n2 - n.
    • T(n) is O(n2)

Insertion Sort processes data one by one and will shift left until the data element is in the correct sorted order. Basically just makes comparisons to from the end of the list and then actually places the element when the element is compared as more than the element to the left of it or the front of the list is reached.
  • What input of data would minimize the repetitions (most efficient)?
    • When the input data is already sorted it will not repeat at all.
    • Only goes through the code once so T(n) = n - 1.
    • T(n) is O(n)
  • What input of data would maximize the repetitions (least efficient)?
    • Data sorted in reverse sorted order as the algorithm would have to compare to the entire array of already sorted data before adding an element to the front.
    • If the data is in reverse order then T(n) = n(n - 1)⁄2 = n2 - n⁄2.
    • T(n) is O(n2)

Selection Sort finds the minimum of the unsorted data set and swaps it with the first unsorted element. This continues until the data is sorted.
  • What input of data would minimize the repetitions (most efficient)?
    • When the input data is already sorted it will not repeat at all.
    • Still makes the same number of comparions though it does not swap the data reuslting in T(n) = n(n - 1)⁄2 = n2 - n⁄2.
    • T(n) is O(n2)
  • What input of data would maximize the repetitions (least efficient)?
    • Data sorted in reverse sorted order as the algorithm would have to compare to the entire array of already sorted data before adding an element to the front.
    • If the data is in reverse order then T(n) = n(n - 1) = n2 - n.
    • T(n) is O(n2)

Non-Comparison Based Sorting

Non-Comparison Based Sorting distributes values to intermediate data structures to help with sorting. It does not make comparisons.

Counting Sort takes an input array of size n integer elements in a range k. Processes each integer input value and counts the number of times each element appears in the input. Creates the sorted output by using frequencies.
  • Steps:
    1. Compute the range.
    2. Create an array C to store frequencies.
    3. Process each value and increment frequencies as necessary.
    4. Accumulate the frequencies.
    5. Use the accumulated frequencies to produce the sorted output by iterating left to right.
  • T(n) is O(n + k)
Radix Sort uses N = radix (the number of unique digits used to represent numbers; base radix = 10). w is the word size of integers (# of digits in an integer). Starts with the least significant digit of each element and adds these elements to buckets. Moves the entries from the buckets back to the main array.
  • Uses counting sort with a range (k) equal to N
  • T(n) is O(n * w)
