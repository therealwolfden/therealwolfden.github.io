Binary Search Trees and Balancable Trees

Binary Search Trees

Binary Search Trees have comparable elements sorted into a tree ADT
  • Can be empty (null).
  • Have a root node R such that...
    • every element of R's left subtree contains data "less than" R's data.
    • every element of R's right subtree contains data "greater than" R's data.
    • R's left and right subtrees are also binary search trees.
  • Leaves are typically sentinel (null) nodes.

search(tree, key, position)
  • The worst case runtime of search in a binary search tree depends on the height of the tree.
Algorithm search(T, k, p)
Input a binary search tree, T
a target key, k
a position, p
Output the last position visited when searching for p
if T.isLeaf(p) then
return p
result ← compare(k, p.getElement().getKey())
if result = 0 then
return p
if result < 0 then
return search(T, k, T.left(p))
return search(T, k, T.right(p))

get(key)
  • Uses the search method to find the key's position and then returns the value at the searched position.
  • The worst case runtime of get in a binary search tree depends on the height of the tree.

put(key, value)
  • Uses the search method to find the key's position.
    • If the position is a sentinel then that position is replaced with the value. The position's children are then made sentinel nodes.
    • If the position is a data value already then that value is replaced with the put parameter and the original value is returned.
  • The worst case runtime of put in a binary search tree depends on the height of the tree.

remove(key)
  • Uses the search method to find the key's position.
    • If the position is a sentinel then the method returns null.
    • If the position is a data value the key is in the tree.
  • Remove operation depends on number of children of the position returned.
    • If the position has 2 sentinel children then remove the position and replace with a sentinel.
    • If the position has 1 non-sentinel child then replace the position with the non-sentinel child.
    • If the position has 2 non-sentinel children then replace with minimum from the right (in order successor).
      • Can be replaced with the maximum from the left but for consistency we replace with the post order successor.

Runtimes
  • For all methods
    • Worst case: O(n)
      • Happens when the tree is basically just a linked list.
    • Best case: O(log(n))
      • Happens when the tree is balanced.

Balanced Trees

Balanced Trees have children spread evenly across their respective roots. This ensure a minimum height.

Balancing Unbalanced Trees
  • Trinode Restructuring
    • Consider node x, its parent y, and its grandparent z.
    • Label the three nodes with a, b, and c as they appear (inorder) from left-to-right.
    • Rotate b (the middle value) to become the new root of the subtree.
    • Ensure subtrees remain in sorted order (those on b may need to be shifted).
    • Each of these rotations reduces the height by 1.

Types of Balenced Trees
  • AVL Trees
    • Binary search trees such that, for every node v, the heights of the children of v differ by at most 1.
  • Splay Trees
    • Binary search trees with new algorithms for map operations.
  • 2-4 Trees
    • Generalization of binary search trees.
  • Red-Black Trees
    • Binary tree representations of 2-4 trees that use red/black color for nodes
