Map ADT

Map Abstract Data Type

Map ADT
  • Models a searchable collection of key-value entries
  • Has "put" (insert), "get" (lookUp), and "remove" (delete) functions
  • Multiple entries with the same key are not allowed (at least in Java)

Key-value Pairs in Java can be implemented using generics as an entry (K, V) that contains a key field and a value field

Functions
  • put(k, v)
    • Checks for duplicates and replaces duplicates if they exist.
  • get(k)
    • Searches for a value with a given key.
  • remove(k)
    • Removes a value with the corresponding key.
  • size()
    • Returns the size.
  • isEmpty()
    • Returns true if the list is empty (size = 0).

List Based Data Structures with Sequential Search

Sequential Search is a search method that goes through the list one at a time.

Array-Based List Performance:
  • put(k, v)
    • O(n)
  • get(k)
    • O(n)
  • remove(k)
    • O(n)

Linked List Performance:
  • put(k, v)
    • O(n)
  • get(k)
    • O(n)
  • remove(k)
    • O(n)

Heuristic is a rule of thumb that results in good performance but not neccessarily optimal. They do not change the worst-case runtime but do lower the expected run time.
  • Move-To-Front Heuristic
    • Whenever an entry is accessed it is moved to the front.
    • New entries are inserted at the front of the list
    • Still traverses linearly but then the most commonly accessed data types are at the front and thus more efficiently found.
  • Transpose Heuristic
    • Whenever an entry is accessed it is moved closer to the front by 1 (switched with the entry before it).
    • New entries are inserted at the front of the list
    • Still traverses linearly but then the most commonly accessed data types are slowly moved to the front and thus become more efficient to find.


Introduction to Data Structures with More Efficient Searching

Comparable Keys allows for the data to be sorted into a particular order allowing for searching to be more efficient.
  • Keys follow a linear order.
  • Entries are listed in increasing/decreasing order based on the key.
  • Data Structures
    • Contiguous-Memory model (search table)
    • Linked-Memory model (skip list)

Search Table

Search Tables are a map implementation using a sorted array-based list. They are great for maps that are mainly used to look up values and not really add or remove them.

Using Binary Search Performance:
  • put(k, v)
    • O(n)
  • get(k)
    • O(log(n))
  • remove(k)
    • O(n)

Skip List

Skip Lists are a map implementation using a sorted linked list. Approximates binary search by trading memory space for speed. Allows for an expected O(log(n)) for insert and remove operations.
  • Contains a series of linked lists (S0, S1, S2, Sh)
  • S0 contains all entries in increasing order by key.
  • Each list is a subsequence of the previous one.
  • Contains quad nodes to store the previous, next, below, above, and the actual data.

Search Operation
  • Starts at the sentinel head (Sh)
  • At the current node compare the key to the next node's data.
    • k = key in the next node: return element(next(p))
    • k > key in the next node: scan forward at the same level
    • k < key in the next node: drop down to the level below
  • Attempting to drop down past the bottom list should return null (no element in the list).

Remove Operation
  • Search for k -> locate positions on all levels with key k.
  • Remove entries with key k from all levels.
  • If you end up with more than one list that contains only the two sentinel head/tail nodes, delete all but one.

Put Operation
  • Search for k and if entries with k already exist, update the values associated with each, then return/stop. If no entries with k already exist, add k to the bottom list S0 and continue
  • Uses a random number generator (RNG) to decide if the node is added above the current list. This repeats until the random number generator generates a heads (50% chance to stop).
  • If the sentinel list gains a node, create a new one to the top of the list.

Skip List Performance

Expected space usage is O(n)

Using the Average/Expected Performance:
  • put(k, v)
    • O(log(n))
  • get(k)
    • O(log(n))
  • remove(k)
    • O(log(n))
