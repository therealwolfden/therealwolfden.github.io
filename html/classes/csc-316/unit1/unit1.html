Problems, Algorithms, and Programs

Problems

Problem: a general question to be answered with given parameters.

Problem Specification: general description of all its parameters and what properties a solution must satisfy.

Problem Instance: provides specific values for all problem parameters.

Traveling Salesperson Problem:
  • Problem: What is the shortest route I should take to visit all cities and return home without visiting a city more than once?
  • Parameters:
    • A finite set of cities C = {C1, C2 , ... , Cn}
    • The distance between each pair of cities (ci, cj)
  • Solution:
    • An ordering of the n cities that minimizes the distance traveled.

Algorithms

Algorithms: a set methodology of finite steps that allows for a solution to be generated for a problem.

Types of Algorithms:
  • Greedy Approach
    • Takes the shortest path that is immediately available.
    • Easy to follow but often does not result in finding the best possible solution.
    • Not a true algorithm
  • Exhaustive Approach
    • Looks at all available paths and then executes the most effective path.
    • Harder to follow and more time consuming but guarantees the best possible solution.
    • Is a true algorithm
Properties of Algorithms:
  • Correctness, always gives the right answer
  • Termination, no infinite loops
  • Robustness, able to handle unexpected inputs
  • Complexity of Resulting Program, easily translates into code
  • Adaptility, able to adapt to small changes in problem specifications
  • Efficiency, low utilization of computational resources

Notation:
  • Assignment: ←
  • Comparisons: <, >, ≤, ≥, =
  • Definite Loop: for i ← X to Y do
  • For-each Loop: for each X in Y do
    • Loop bounds are inclusive (for i ← 1 to n-1 do is equal to for(int i = 1; i <= n-1; i++))
  • If conditional: if X then
  • Indefinite Loop: while X do
  • Return: return x
  • Comment: //

Efficiency and Data Structures

Efficiency:
  • Time efficient algorithms can be applied to large scale problems and focus on solving a problem as quickly as possible.
  • Space (memory) efficient algorithms are largely used for small devices such as phones and prioritize taking up as little memore as possible.
Data Structures often influence the time and memory efficiency of an algorithm. They are systematic ways of organizing data.
  • Static Data Structure: Cannot be changed without recreating it from scratch such as an array.
  • Dynamic Data Structure: May increase and decrease size at will such as a linked list.
  • Deterministic Data Structure: A sequence of insert, find, and delete operations with key values which will always result in the same resulting structure.
  • Probabilistic Data Structure: Same sequence of operations which do not result in the same data structure and are typically based on a randomization component.

Programs

Computer Programs: an implementation of an algorithm. Typically written in coding languages such as Java, Python, etc.
