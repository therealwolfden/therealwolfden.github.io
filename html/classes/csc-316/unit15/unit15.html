Hash Tables: Hash Functions

Hashing

Hashing uses the concept of key transformation.
  • Provides fast implementations of map ADTs.
  • Stores n items in an array or Hash Table with array capacity m.
  • Use a hash function to store items with key k in position h(k) of the hash table.

Hash Functions, h(k), transform any key into one of m << k positions.
  • Involves a hash code and a compression function.
  • Should be easy to compute/execute due to their frequent usage during implementation.
  • Should spread out keys uniformly to minimize collisions.
    • Collisions are when two objects have the same hash code value but do not equal each other.
    • Needs to minimize them as well as resolve possible collisions if they can't be avoided.

Hash Codes are integer values corresponding to a key.
  • May provide a value not within the range (≥ m) or even a negative value.
  • A set of hash codes should eliminate collisions as often as possible.
  • There are several hash code schemes.
    • Additive Hash Code
    • Polynomial Hash Code
    • Cyclic-shift Hash Code

Hash Code Schemes

Additive Hash Code
  • Generates a numerical hash code equal to the sum of all corresponding ascii values of a string.
  • A string "ABC" would hash to (65 + 66 + 67 = 198).
  • While this is easy and effective it results in collisions often due to it not being dependent on order.

Polynomial Hash Code
  • Generates a numerical hash code equal to the sum of all ascii values multiplyed by ridx where r is a nonzero constant.
  • A string "ABC" with r = 33 would hash to (65 * 330 + 66 * 331 + 67 * 332= 75206).
  • Common values of r to minimize collisions are 33, 37, 39, or 41 as they experimentally result in < 7 collisions.
  • This method accounts for order and greatly reduces collisons while still being relatively easy.

Cyclic-shift Hash Code
  • Replaces the multiplication in polynomial hashing.
  • Shifts the first 5 bits in a 32 bit value to the end.
  • Uses an unsigned right shift so that 0s are always used to pad the front.
  • Shifting by 5 is experimentally determined to be a shift value that reduces collions the most.
Algorithm cyclicShift(S)
Input a string S
Output the hash code for S
h ← 0
for i ← 0 to length(S) - 1 do
h ← (h << 5) | (h >>> 27)
h ← h + charAt(S, i)
return h

Compression Functions

Compression Functions:
  • Converts an integer key value to a value within the range of 0 < val < m.
  • Compression Strategies.
    • Division method
      • Takes the hash code and mods by m [h(k) = f(k) mod m].
      • r-1 should not be evenly divisible by m. In most cases m should be a prime number.
    • Multiply-and-Divide (MAD) Method
      • h(k) = (αf(k) + β) mod m α > 0, β ≥ 0
      • α and β are constants where α mod m ≠ 0.
    • Golden-Ratio Method
      • Allows m to be any integer value instead of just prime.
      • φ = 1 + √(5) ⁄ 2 = 1.61803399...
      • 1 ⁄ φ = φ - 1 = 0.61803399...
      • h(k) = floor(m{f(k)φ-1})
      • For a large n, the n fractions iφ − 1 are distributed almost uniformly in (0,1)

Hashing Utility

Hashing is based off the equals() method
  • The values hashed within an object are based off the fields compared in equals().
  • If two objects are considered equal, regardless on if they have different fields, they should have the same hashcode.
  • Use a hash function to store items with key k in position h(k) of the hash table.

Hashing in Security
  • Hashing is easy to generate but hard to go back to the original value (One-way Function).
  • Commonly used when values need to be compared without knowing the original value.
  • Passwords are commonly hashed to be stored and compared.

SHA-1 or Secure Hash Algorithm 1
  • Provides a 160-bit value called a message digest.
  • Designed by the NSA.
  • No longer considered secure as it is susceptible to collision attack.
  • Git uses it to identify revisions.

Cracking Hashing
  • Dictionary Attacks uses a file of common passwords and other strings that are likely to be used as passwords.
  • Brute Force Attacks tries every possible combination of characters.
  • Lookup Tables Attacks use pre-computed hashes of common strings to find a password.
  • To prevent cracking many security hashes use a randomly generated "salt" or random value in the hash. This random salt is stored in the database as well.
