List and Positional List ADTs

Abstract Data Types

Abstract Data Types (ADTs) model behaviors from the point of view of a user of the data.
  • Outlines the possible operations of a data type.
  • Does not describe how operations are implemented, only their functionality.
  • No rule for what operations must be supported.
  • First described by Barbara Liskov.
  • ADT behaviors should be used when documenting algorithms or for pseudocode.

Data Structures

Data Structures are concrete implementations of ADTs
  • Implement operations defined in the ADT.
  • Different types of Data Structures have their own benefits for use with an ADT.
  • List Data Structure
    • Array-based List
      • Contiguous memory representation.
      • Adjacency in the array means adjacency in the list.
    • Linked List
      • Linked memory representation.
      • Elements can be scattered arbitrarily in memory.
      • List elements store references to neighbor(s).

Array-based List

Array-based List:
  • Benefits
    • Access to each element in constant O(1) time.
  • Problems
    • Array size is fixed and thus can not dynamically resize causing wasted memory.
      • Array must grow when the elements in the list exceed the length of the array.
    • Inserting/removing from the middle causes massive data movement with O(n) time.

Armortized Analysis

Armortized Cost:
  • Gradually reduce the cost of an operation over a sequence of operations.
  • Basically the idea that some expensive operations may allow for future operations to be much cheaper in terms of memory/runtime.

Armortized Analysis:
  • The armortized analysis is stated in terms of the pay off. An armortized cost of O(1) says that over a series of operations the cost is O(1) and can not be applied on an individual use of the operation.
  • An armortized of cost O(1) is assigned to adding a value to the end of an array-based list. Therefore the total running time of adding n values to the end of an array-based list is O(n)

Linked List

Linked List:
  • Benefits
    • Insertions/deletion do not involve moving or shifting.
    • Dynamic allocation of memory on a as-needed basis.
  • Problems
    • No random access meaning accessing data in the middle of the list is in O(n) time.

Singly Linked List:
  • Consists of Listnodes that store data and point to the next node within the list.
  • A head pointer leads to the front of the list.
  • Some implementations have a tail pointer that points to the back of the list.

Circular Linked List:
  • The head pointer still points to the start of the list.
  • The tail pointer now points to the head of the list instead of an empty/null node.

Doubly Linked List:
  • Consists of Listnodes that store data and point to the next and previous nodes within the list.
  • Allows for easy insertions and deletions at the cost of extra memory usage.

Sentinel Node:
  • No data stored within the node (null).
  • The next points to the true beginning of the list.
  • Allows for less special cases and a more uniform code.

Positional List:
  • Special abstraction of the list ADT that manipulates data using the location/position.

    • Array size is fixed and thus can not dynamically resize causing wasted memory.
      • Array must grow when the elements in the list exceed the length of the array.
    • Inserting/removing from the middle causes massive data movement with O(n) time.

Iterators

Iterators provide a common, efficient way to traverse many types of data structures.
  • Think of an iterator as "between" two elements and points to the element after it.
  • Iterators can remove the next element as well as well iterate or go step by step through the data set.
  • In pseudocode iterators can be implyed by using a "for-each" style loop.
